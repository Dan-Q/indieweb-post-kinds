<a id="user-content-indieweb-post-kinds" class="anchor" href="#indieweb-post-kinds" aria-hidden="true"><span class="octicon octicon-link"></span></a>indieweb-post-kinds</h1>

<p>Indieweb Post Kinds adds a Custom Taxonomy to the standard post type in Wordpress that allows posts to have a semantic component, based on the same conventions as the built-in Post Formats.</p>

<p>While the conventions are the same as post formats and thus can be used for theming, post kinds are less about formatting and more about the display and markup of context.</p>

<p>This allows archives of replies, likes, reposts, etc. as well as adding the  appropriate classes to links within same.</p>

<p>The original version of this remains as Indieweb Taxonomy. Some of the kinds will not be used by everyone...
in future, there will be an option to hide/select the active ones in the post editor.</p>

<h2>
<a id="user-content-post-kinds" class="anchor" href="#post-kinds" aria-hidden="true"><span class="octicon octicon-link"></span></a>Post Kinds</h2>

<h3>
<a id="user-content-the-non-response-kinds" class="anchor" href="#the-non-response-kinds" aria-hidden="true"><span class="octicon octicon-link"></span></a>The Non-Response Kinds</h3>

<ul class="task-list">
<li>
<strong>Article</strong> - traditional long form content</li>
<li>
<strong>Note</strong> - short content</li>
<li>
<strong>Photo</strong> - image or photo post</li>
</ul>

<h3>
<a id="user-content-the-response-kinds" class="anchor" href="#the-response-kinds" aria-hidden="true"><span class="octicon octicon-link"></span></a>The Response Kinds</h3>

<ul class="task-list">
<li>
<strong>Reply</strong> - Replying to someone else's comment</li>
<li>
<strong>Repost</strong> - a complete repost of someone else's content</li>
<li>
<strong>Like</strong> -  props/compliments to the original post/poster</li>
<li>
<strong>Favorite</strong> - special to the favoriter</li>
<li>
<strong>Bookmark</strong> - also known as a linkblog. This is basically sharing/storing a link/bookmark.</li>
<li>
<strong>Tag</strong> - Allows you to tag a post as being of a specific tag, or person tagging.</li>
<li>
<strong>RSVP</strong> - A specific type of Reply regarding an event (Not Fully Fleshed Out)</li>
</ul>

<h3>
<a id="user-content-future-kinds-possibly" class="anchor" href="#future-kinds-possibly" aria-hidden="true"><span class="octicon octicon-link"></span></a>Future Kinds (Possibly)</h3>

<ul class="task-list">
<li>
<strong>Check-In</strong> - Identifying you are at a place</li>
</ul>

<h2>
<a id="user-content-future-plans" class="anchor" href="#future-plans" aria-hidden="true"><span class="octicon octicon-link"></span></a>Future Plans</h2>

<ul class="task-list">
<li>Add H-Card/Author Support using the functions in H-Card Tools Plugin</li>
<li>Custom Meta Box for Kinds to replace the generic class</li>
<li>Automatic import/parsing of information based on click of button</li>
<li>Contextual response box, hiding/changing options based on Kind selected. Example, an RSVP that shows Yes/No/Maybe.</li>
</ul>

<h2>
<a id="user-content-functions" class="anchor" href="#functions" aria-hidden="true"><span class="octicon octicon-link"></span></a>Functions</h2>

<p><code>get_post_kind($id)</code> - Return the kind for a given post. If <code>$id</code> is not specified, use current post.</p>

<p><code>has_post_kind($kind, $post)</code> - Returns true/false if kind is in post. If post is empty, then use current post</p>

<p><code>get_kind_context_class( $class, $classtype )</code> - Returns the CSS class to be applied to the response/context if the kind is one for which there is context. Classtype defaults to u, other option is usually p. Sets the class to the kind slug and for specially specified slugs, sets appropriate mf2 classes as well. $class specifies any additional classes to be added.</p>

<h2>
<a id="user-content-filters" class="anchor" href="#filters" aria-hidden="true"><span class="octicon octicon-link"></span></a>Filters</h2>

<p><code>get_the_kind</code> - Filter get_the_kinds</p>

<p><code>the_kinds</code> - Filter get_the_kinds_list</p>

<p><code>kind_classes</code> - Filter get_kind_class</p>

<p><code>kind_verb</code> - Filter get_kind_verbs</p>

<p><code>response_display</code> - Filters the output being added to the_content or to custom location in theme</p>

<a id="user-content-changelog" class="anchor" href="#changelog" aria-hidden="true"><span class="octicon octicon-link"></span></a>Changelog</h2>

<ul class="task-list">
<li><p><em>Version 0.22</em> - Complete response html cached to post-meta and purged on post update.</p></li>
<li><p><em>Version 0.21</em> - Temporary fix for slow embed code in preparation for caching to reduce calls. Removed defaultterms check to only run on settings page save or plugin activation instead of on each load.</p></li>
<li><p><em>Version 0.2</em> - Forked to Indieweb Post Kinds to reflect a change to act as an alternative to the post formats feature of WordPress. Removed multikind option. Prior to this, the plugin functions mirrored those of tags. With the removal of multikind support, each post can only have one kind, and the functions will more closely mirror the Post Format. Complete rewrite of the display functionality.</p></li>
<li><p><em>Version 0.11</em> - Option to Update Metadata Deliberately Commented Out. Anyone who used the old plugin should backup their database before considering migration. The data should still be in the database regardless.</p></li>
<li><p><em>Version 0.1</em> - Revised settings page to use WordPress Settings API. Alert: This version switches to using an array for storage of response data for future development. To migrate your data, please backup your database and then check "Migrate to new data structure on update" on the options page.</p></li>
<li><p><em>Version 0.06</em> - Added in support for Favorite, fixed webmention support.</p></li>
<li><p><em>Version 0.05</em> - Set up embed handler for commonly linked sites. Add options to turn on embeds for these sites. Plugin is probably now feature complete enough for test deployment to a live site.</p></li>
<li><p><em>Version 0.04</em> - Add function to generate verbs(Like to Liked...) for a given kind. Add Display functionality in Beta. Include Genericons and Dashicons for icon options for the various kinds. Add plugin option to add the response URL to the top or bottom of the content section. Plugin near point at which can be deployed for basic use.</p></li>
<li><p><em>Version 0.03</em> - Location function removed and split into separate plugin called Simple Location. Any check-in kind functionality will have this as a dependency.</p></li>
<li><p><em>Version 0.02</em> - Location meta box with HTML5 geolocation fill-in added. This allows posts to optionally have a location. This is as per the Wordpress Geodata specifications, so the Wordpress Android app will fill them in. There is no display functionality. Various functions that mimic the built-in functions for other taxonomies were added, including filters to add additional behaviors. Default terms now prepopulate if no terms exist.</p></li>
<li><p><em>Version 0.01</em> - Registers a custom taxonomy, adds in code snippets to turn the post meta box from checkboxes to radio buttons, adds code to allow a custom permalink tag if needed.</p></li>
</ul>
